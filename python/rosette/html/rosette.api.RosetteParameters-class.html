                      RosetteParameters

Parameter object for all operations requiring input other than translated_name. Four fields, content, contentType, unit, and inputUri, are set via the subscripting operator, e.g., params["content"], or the convenience instance methods RosetteParameters.LoadDocumentFile and RosetteParameters.LoadDocumentString. The unit size and data format are defaulted to InputUnit.DOC and DataFormat.SIMPLE.

Using subscripting instead of instance variables facilitates diagnosis.

If the field contentUri is set to the URL of a web page (only protocols http, https, ftp, ftps are accepted), the server will fetch the content from that web page. In this case, neither content nor contentType may be set.

LoadDocumentFile(self, path, dtype)

source code 

Loads a file into the object. The file will be read as bytes; the appropriate conversion will be determined by the server. The document unit size remains by default InputUnit.DOC.

Parameters:
  • path - Pathname of a file acceptable to the open function.
  • dtype (DataFormat) - One of DataFormat.HTML, DataFormat.XHTML, or DataFormat.BINARY. No other types are acceptable at this time, although HTML is broad enough to include text strings without markup. If the data type is unknown, or describes a binary file, use DataFormat.BINARY.

LoadDocumentString(self, s, dtype)

source code 

Loads a string into the object. The string will be taken as bytes or as Unicode dependent upon its native python type and the data type asked for; if the type is HTML or XHTML, bytes, not python Unicode, are expected, the encoding to be determined by the server. The document unit size remains (by default) InputUnit.DOC.

Parameters:
  • s - A string, possibly a unicode-string, to be loaded for subsequent analysis, as per the dtype.
  • dtype (DataFormat) - The data type of the string, as per the enum DataFormat.
